Operators

The Ternary Operator ?: is basically acting like an "if X is true, then Y otherwise Z" 
const x = undefined
console.log(x ? "is defined" : "is undefined") // -> "is undefined"
const luck = 888
console.log(luck > 100 ? "much luck" : "could be better") // -> "much luck"
Due to the logical OR returning the right side of the || if the left is false, you can use it for defaulting values 
const x = null
const y = x || "not null"
console.log(y) // -> "not null"
But due to 0 and '' returning false this may lead to unintended behavior and instead of writing explicit null/undefined checks, never JavaScript has the nullish coalescing operator ?? 
const x = 0
// that's bad, we only want defaults for when it's actually undefined/null
console.log(x || "some default") // -> some default
// too long
const y = (x !== null && x !== undefined) ? x : "some default";
// use
const y = x ?? "some default"
