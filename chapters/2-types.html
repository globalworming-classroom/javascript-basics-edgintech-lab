Types

values always have a type that can be checked with the typeof operation 
const truth = false;
const quote = 'hello';
const count = 1;
const infinite = Infinity // e.g. when calculating 1 / 0
const notANumber = NaN // e.g. when calculating 0 / 0
const thing = {};
const emptyArray = [];
const undefinedValue = undefined;
const nullValue = null;
console.log(typeof truth) // boolean
console.log(typeof quote) // string
console.log(typeof count) // number
console.log(typeof infinite) // number
console.log(typeof notANumber) // number
console.log(typeof thing) // object
console.log(typeof emptyArray) // object, arrays are special objects
console.log(typeof undefinedValue) // undefined
console.log(typeof null) // object,  that's just one instance of the weird things JS does
When working with values of different types, Javascript will make best guess on which types you want. That's called type coercion 

console.log(typeof (quote + count)) // string
console.log(typeof (truth  + count)) // number
You can reassign variables no matter the type 

let agile = true
console.log(typeof agile) // boolean
agile = 99.99
console.log(typeof agile) // number
